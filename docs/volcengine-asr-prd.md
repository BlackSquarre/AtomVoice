# PRD: 豆包云端语音识别引擎接入

> **状态标记**: ✅ 已完成 | 🔧 部分完成 | ⏳ 待实现 | ❌ 未开始

## 1. 背景

AtomVoice 目前支持两类语音识别引擎：Apple Speech Recognition 和 Sherpa-ONNX 本地模型。Apple 引擎系统集成度高，但存在系统服务限制、语言支持和稳定性差异；Sherpa-ONNX 隐私性最好，但当前模型偏中文且识别质量、标点、长句稳定性有限。

火山引擎"豆包语音"提供大模型流式语音识别 API，支持 WebSocket 实时识别、标点、ITN、多语言和二遍识别能力。接入后可作为可选的云端识别引擎，为用户提供更高准确率和更稳定的长语音体验。

## 2. 目标

1. ✅ 在 AtomVoice 中新增一个可选识别引擎：豆包云端 ASR。
2. ✅ 保持默认隐私策略不变：默认仍使用本地或系统识别，云端识别必须由用户主动启用。
3. ✅ 复用现有 Fn 录音、胶囊 UI、自动停止、LLM 润色和文本注入流程。
4. ✅ 在录音过程中实时展示豆包 ASR 的部分结果，停止录音后注入最终文本。
5. ✅ 提供清晰的配置入口、隐私提示和错误反馈。

## 3. 非目标

1. 不在首版实现火山热词表、自学习词表、说话人分离、情绪识别、性别识别、语速和音量分析。
2. 不在首版实现离线缓存或失败后自动重放音频到其他引擎。
3. 不改变 Apple 和 Sherpa-ONNX 现有行为。
4. 不把豆包 ASR 作为默认识别引擎。
5. 不在首版提供企业级账号、多项目、多 API Key 管理。

## 4. 用户场景

### 4.1 高准确率中文听写 ✅

用户希望中文长句、口语断句、标点和数字规范化更准确，可在菜单中选择"豆包云端 ASR"，按住触发键说话，松开后将识别文本注入当前输入框。

### 4.2 多语言听写 ✅

用户切换 AtomVoice 识别语言后，豆包 ASR 使用对应 language 参数，例如 `zh-CN`、`en-US`、`ja-JP`、`ko-KR`、`es-MX`、`fr-FR`、`de-DE`。

### 4.3 云端识别加 LLM 润色 ✅

用户启用豆包 ASR 和现有 LLM Refinement。录音结束后，豆包返回的最终文本进入现有 LLM 润色流程，随后注入文本。

### 4.4 隐私敏感用户 ✅

用户不启用豆包 ASR 时，应用仍保持现有本地优先体验。首次选择豆包 ASR 时，应用明确提示音频会发送到火山引擎服务。

## 5. 产品范围

### 5.1 识别引擎菜单 ✅

新增识别引擎选项：

| 引擎 | 说明 |
| --- | --- |
| Apple Speech Recognition | 现有系统引擎 |
| Sherpa-ONNX Local | 现有本地引擎 |
| 豆包云端 ASR | 新增火山引擎 WebSocket 流式识别 |

用户选择豆包云端 ASR 时，如果未配置 API Key，应打开设置或显示配置提示，不直接开始录音。

### 5.2 设置项 ✅

新增豆包 ASR 设置区域，建议包含：

| 设置项 | 类型 | 默认值 | 说明 | 状态 |
| --- | --- | --- | --- | --- |
| 启用豆包云端 ASR | 引擎选择 | 关闭 | 通过识别引擎菜单选择 | ✅ |
| API Key | 密码输入 | 空 | 新版控制台 `X-Api-Key` | ✅ |
| Resource ID | 文本或下拉 | `volc.seedasr.sauc.duration` | 默认使用 ASR 2.0 小时版资源 ID | ✅ |
| Endpoint | 文本或高级设置 | `wss://openspeech.bytedance.com/api/v3/sauc/bigmodel_async` | 默认使用双向流式优化接口 | ✅ |
| 启用标点 | 开关 | 开 | 对应 `enable_punc`，跟随全局设置 | ✅ |
| 启用 ITN | 开关 | 开 | 对应 `enable_itn` | ✅ |
| 启用顺滑 | 开关 | 关 | 对应 `enable_ddc` | ✅ |
| 文本润色（二遍识别） | 开关 | 关 | 对应 `enable_nonstream` | ✅ |

高级设置可先隐藏，首版 UI 可以只暴露 API Key 和 Resource ID，其余使用合理默认值。

### 5.3 首次启用提示 ✅

首次选择豆包云端 ASR 时展示确认提示：

| 内容 | 要求 |
| --- | --- |
| 隐私说明 | 使用该引擎时，录音音频会发送到火山引擎进行识别 |
| 费用说明 | 火山引擎可能产生用量费用，具体以用户账号配置为准 |
| 操作 | "继续使用"和"取消" |

用户确认后记录同意状态。用户取消时保持原识别引擎不变。

### 5.4 录音体验 ✅

豆包 ASR 启用后，录音交互应与现有引擎一致：

| 动作 | 期望结果 | 状态 |
| --- | --- | --- |
| 按住触发键 | 建立 WebSocket，开始录音和流式发送音频 | ✅ |
| 录音中 | 胶囊展示实时识别文本和波形 | ✅ |
| 松开触发键 | 发送最后一包音频，等待最终结果 | ✅ |
| 最终结果非空 | 进入自动标点、LLM 润色或直接注入流程 | ✅ |
| 最终结果为空 | 胶囊关闭，不注入文本 | ✅ |
| ESC | 取消录音，关闭 WebSocket，不注入文本 | ✅ |
| Space/Backspace | 立即停止并注入当前文本，跳过 LLM | ✅ |

### 5.5 错误反馈 ✅

需要覆盖以下错误：

| 场景 | 用户反馈 | 状态 |
| --- | --- | --- |
| 未配置 API Key | 显示"请先配置豆包 ASR API Key" | ✅ |
| Resource ID 为空 | 显示"请先配置豆包 ASR Resource ID" | ✅ |
| WebSocket 连接失败 | 显示网络或鉴权错误摘要 | ✅ |
| 服务端错误帧 | 显示错误码和短错误信息 | ✅ |
| 超时无结果 | 显示"识别超时，请重试" | ✅ |
| 音频格式错误 | 显示"音频格式不被服务端接受" | ✅ |
| 用户取消 | 不显示错误，不注入文本 | ✅ |

### 5.6 权限与隐私 🔧

1. ✅ 豆包 ASR 必须需要麦克风权限。
2. ✅ 文本注入仍需要辅助功能权限。
3. ⏳ 豆包 ASR 不依赖 macOS Speech Recognition 权限。
4. ⏳ 权限窗口应避免误导：当用户只使用豆包 ASR 时，Speech Recognition 权限不是必需项。
5. ⏳ README 和隐私说明在功能发布前必须更新，说明云端 ASR 会上传音频。

### 5.7 本地化 ✅

所有新增用户可见字符串必须走 `loc()`，并同步更新全部 lproj：`en`、`zh-Hans`、`zh-Hant`、`ja`、`ko`、`es`、`fr`、`de`。

## 6. 成功指标

1. ✅ 用户可以从菜单选择豆包云端 ASR，并完成一次完整录音注入。
2. ✅ 中文普通话 10 秒以内短句可以稳定返回最终文本。
3. ✅ 录音中可看到至少一次实时部分结果更新。
4. ✅ API Key 缺失、鉴权失败、网络失败都有明确错误提示。
5. ✅ 未启用豆包 ASR 的用户体验不发生变化。

## 7. 验收标准

### 7.1 功能验收 ✅

1. ✅ 配置有效 API Key 和 Resource ID 后，选择豆包云端 ASR 可以开始录音。
2. ✅ 胶囊实时显示豆包返回的识别文本。
3. ✅ 松开触发键后，最终文本被注入当前输入框。
4. ✅ 启用 LLM Refinement 时，豆包文本进入现有润色流程。
5. ✅ ESC 取消不会注入文本，也不会留下悬挂 WebSocket 连接。
6. ✅ Space/Backspace 立即停止时，跳过 LLM 并注入当前文本。

### 7.2 兼容性验收 ✅

1. ✅ Apple 引擎仍可使用滚动分段和 Live Insertion。
2. ✅ Sherpa-ONNX 引擎仍可下载、加载、识别和标点。
3. ✅ 无 API Key 的用户启动应用不受影响。
4. ✅ 离线环境下选择豆包 ASR 会提示网络失败，不影响切回本地引擎。

### 7.3 隐私验收 🔧

1. ✅ 首次选择豆包 ASR 时必须展示云端上传提示。
2. ✅ 用户取消提示后，不切换到豆包 ASR。
3. ✅ 日志不得打印 API Key 或完整鉴权 header。
4. ✅ API Key 应存储在 Keychain。

## 9. 待确认问题

1. 默认 Resource ID 使用 `volc.seedasr.sauc.duration` 还是由用户必须手动填写？
2. 是否必须支持旧版控制台的 `X-Api-App-Key` 和 `X-Api-Access-Key`？建议首版只支持新版 `X-Api-Key`。
3. 是否在设置页提供"测试连接"按钮？建议提供，但可放到第二阶段。
4. 是否默认开启二遍识别 `enable_nonstream`？建议开启，以优先保证最终文本准确率。
5. 是否允许豆包 ASR 与 Apple Live Insertion 同时工作？建议首版不支持，避免重复注入和结果回滚问题。

## 10. 多厂商云端 ASR 扩展规划

豆包 ASR 上线后，可扩展接入其他云端 ASR 厂商。以下是主流厂商的流式识别 API 对比。

### 10.1 厂商对比

| | 火山引擎（豆包） | 阿里云 | 腾讯云 | 科大讯飞 |
|---|---|---|---|---|
| **协议** | WebSocket 二进制帧 | WebSocket JSON+Binary | WebSocket Binary | WebSocket Binary |
| **鉴权** | Header（X-Api-Key） | Token | HMAC-SHA1 签名 | HMAC-SHA1 签名 |
| **音频格式** | PCM/WAV/OGG/MP3 | PCM/WAV/OPUS/MP3 等 | PCM/WAV/OPUS/SILK 等 | 仅 PCM |
| **音频分片** | 200ms | 3200 字节 | 200ms / 6400 字节 | 40ms / 1280 字节 |
| **结束信号** | 二进制帧 flags | JSON 指令 | `{"type":"end"}` | `{"end":true}` |
| **响应格式** | 二进制帧 + JSON | JSON 事件 | JSON（slice_type） | JSON（嵌套） |
| **中间结果** | ✅ | ✅ | ✅ | ✅ |
| **标点** | ✅ | ✅ | ✅ | ✅ |
| **ITN 数字转换** | ✅ | ✅ | ✅ | ✅ |
| **语义顺滑** | ✅ | ✅（disfluency） | ✅（filter_modal） | ✅（顺滑词） |
| **二遍识别** | ✅（enable_nonstream） | ❌ | ❌ | ❌ |
| **语言数** | 20+ | 少（需控制台配置） | 37+ | 中英+方言 |
| **集成难度** | ⭐⭐ 中 | ⭐⭐ 中 | ⭐⭐⭐ 中高 | ⭐⭐⭐ 中高 |

### 10.2 各厂商鉴权方式

| 厂商 | 鉴权方式 | 说明 |
|---|---|---|
| 火山引擎 | HTTP Header | `X-Api-Key` + `X-Api-Resource-Id`，最简单 |
| 阿里云 | URL Token | 先调 API 获取 Token，再拼到 WebSocket URL |
| 腾讯云 | URL 签名 | 对所有参数排序 + HMAC-SHA1 + Base64，较复杂 |
| 讯飞 | URL 签名 | MD5(appid+ts) + HMAC-SHA1(apiKey) + Base64，较复杂 |

### 10.3 推荐接入优先级

| 优先级 | 厂商 | 原因 |
|---|---|---|
| 1 | 火山引擎（豆包） | 已实现，鉴权最简单，二遍识别独有 |
| 2 | 阿里云 | 鉴权较简单（Token），JSON 指令清晰，文档质量好 |
| 3 | 腾讯云 | 语言最多，功能最全，但签名复杂 |
| 4 | 科大讯飞 | 中文老牌，但响应格式嵌套复杂，集成成本最高 |

### 10.4 通用接入流程

用户切换云端 ASR 厂商时的流程：

1. 用户在菜单中选择厂商
2. 首次选择时提示隐私说明（音频会发送到对应厂商）
3. 用户在设置页填写该厂商的凭据
4. 录音时自动使用对应厂商的 WebSocket 连接
5. 松手后等待最终结果，注入文本
