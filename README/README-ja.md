# AtomVoice

<p align="center"><img src="AppIcon-1024.png" width="128"></p>

<h3 align="center">押して、話す。</h3>
<p align="center">軽量・プライバシー優先の音声ディクテーション。文字はそのまま任意の Mac アプリに入力され、録音時間に制限はありません。</p>



---

### 🔒 プライバシー優先
AtomVoice はクラウド処理を明示的な選択にしています。Sherpa-ONNX は完全オフライン、Apple 音声認識は対応言語で端末上認識を強制でき、Doubao Cloud ASR と LLM テキスト最適化は任意で有効化します。AtomVoice 自体はサーバーを運用せず、録音や文字起こし履歴を保存しません。

### ⚡ 軽量設計
アプリ本体は小さく、待機時の CPU 使用率はほぼゼロ、バックグラウンド常駐プロセスもありません。Sherpa のランタイム、ASR モデル、句読点モデルは必要に応じてダウンロードされ、大きなローカルモデルはアイドル時または重大なメモリ圧迫時に解放できます。

---

## 機能

### 録音とトリガー
- **長押し / シングルタップ** の 2 モード、無音自動停止と組み合わせ可能
- **トリガーキーをカスタマイズ** — 自分のキーボードに合った修飾キーを選択
- **録音中ショートカット** — 1 キーでキャンセル、LLM をスキップして即時挿入、句読点で即終了
- **ヘッドフォン音声コントロール (Beta)** — ヘッドフォンの再生/一時停止ボタンで音声入力を操作: シングル押しは入力モードに従い、長押しで録音、ダブル押しで Return を送信
- **アプリ切り替え時に自動キャンセル**(長押しモードのみ)

### 認識エンジン
- **Apple 音声認識** — システムエンジン。ストリーミング、端末上認識オプション、SFSpeechRecognizer の 1 分制限を超える**ローリング分割**に対応
- **Sherpa-ONNX** — 完全オフラインのローカルエンジン。言語別モデルプリセット、CPU/Core ML バックエンド、ランタイム/ASR/句読点モデルのオンデマンドダウンロード、自動アンロード、サードパーティモデルのインポートに対応
- **Doubao Cloud ASR** — 任意の Volcengine ストリーミング認識。API Key はキーチェーン保存、ITN、スマート句読点、テキスト整流、任意の最終パス認識、クラウド失敗時の Apple Speech フォールバックに対応
- **8 種の UI / 認識言語** — English、简体中文、繁體中文、日本語、한국어、Español、Français、Deutsch。Sherpa は内蔵プリセットがない言語でもインポートしたサードパーティモデルを使用可能

### テキスト出力
- **Apple リアルタイム挿入** — 録音中に完成した文をそのまま挿入、キーを離す前に反映
- **スマート句読点** — ローカルのヒューリスティックな句読点エンジン(言語別);カーソル直後にすでに句読点がある場合は自動でスキップ
- **CJK IME 互換** — 貼り付け前に一時的に ASCII レイアウトに切り替え、完了後に復元
- **LLM テキスト最適化 (Beta)** — 認識テキストのみを後処理。OpenAI 互換プロトコルと **Anthropic** に対応、ストリーミングプレビュー、10 種のプリセット + 自由編集可能なカスタムリスト、多言語のデフォルト system prompt または独自 prompt

### UI とアニメーション
- **5 バンド FFT スペクトル波形** — 人声に最適化(100–4200 Hz)、Accelerate ベース
- **3 種のアニメーション** — Dynamic Island(Spotlight 風スプリング + ガウシアンブラー)/ ミニマル / なし、3 段階速度、ProMotion 120Hz 対応
- **Liquid Glass**(macOS 26)/ **Visual Effect ブラー**(macOS 14/15)
- **8 言語の UI**、システム言語に追従

### システム連携
- **初回セットアップ** — 権限、入力モード、認識エンジンの選択を案内
- **自動アップデート** — GitHub Releases から取得、SHA256 とコード署名を検証(Beta チャネルもオプションで)
- **ログイン時に自動起動**(SMAppService)
- **オーディオ入力デバイス選択** — 任意のマイクを指定可能
- **オーディオ経路の復旧** — 録音中にヘッドホン、AirPods、入力デバイスが切り替わっても復旧可能。認識エンジンごとに自動リサンプリング
- **録音中にシステム音量を下げる**(オプション)
- **シングルインスタンス保護** — 起動時に古いインスタンスを自動終了

## 動作要件

- **macOS 14 Sonoma 以降**
- 必要な権限:**アクセシビリティ**、**マイク**、**音声認識**

## インストール

**Release からダウンロード(推奨)**

[Releases](https://github.com/BlackSquarre/AtomVoice/releases) から対応するアーキテクチャの zip をダウンロードし、解凍してアプリケーションフォルダにドラッグ。各リリースで Universal / Apple Silicon / Intel の 3 種類を提供。

**Homebrew**

```bash
brew tap BlackSquarre/tap
brew install --cask atomvoice
```

**ソースからビルド**

```bash
git clone https://github.com/BlackSquarre/AtomVoice.git
cd AtomVoice
make dev
open dist/Test/AtomVoice.app
```

アーキテクチャチェック:

```bash
make test
```

Makefile は `Makefile` 内で設定された Apple Development 署名 ID でアプリをバンドル/署名します。別の Mac でビルドする場合は自分の署名 ID に変更してください。

## ⚠️ 署名について

Apple の公証は受けていません。初回起動時:

1. `AtomVoice.app` を右クリック → **開く** → **開く** をクリック
2. または **システム設定 → プライバシーとセキュリティ** → **このまま開く**
3. またはターミナルで:`xattr -cr /Applications/AtomVoice.app`

## 使い方

| 操作 | 動作 |
|------|------|
| トリガーキー長押し | 録音開始(長押しモード) |
| トリガーキーを離す | 録音終了し文字を挿入 |
| トリガーキーをタップ | 録音の開始 / 停止(タップモード) |
| 録音中にカプセルをクリック | 録音終了し文字を挿入 |
| 録音中に `ESC` | キャンセル、文字は挿入されない |
| 録音中に `Space` / `Backspace` | 即座に挿入、LLM をスキップ |
| 録音中に句読点を入力 | 即座に挿入し、その句読点を付加 |
| ヘッドホンの再生/一時停止ボタン(任意, Beta) | シングル押しはモードに従い、長押しで録音、ダブル押しで Return |
| メニューバーアイコン | エンジン / 言語 / 入力モード / アニメーション / ASR 設定 / LLM の切り替え |

## 認識エンジン設定

- **Apple Speech** はそのまま使えます。選択した言語が対応している場合は端末上認識を有効化できます。
- **Sherpa-ONNX** は **認識エンジン設定 → Sherpa ローカル** で設定します。言語、モデルプリセット、CPU/Core ML バックエンド、自動アンロード時間、またはサードパーティモデルパッケージを選択/インポートできます。
- **Doubao Cloud ASR** は **認識エンジン設定 → Doubao Cloud ASR** で設定します。Volcengine API Key を入力し、モデルバージョンを選び、WebSocket endpoint を保持または編集します。初回切り替え時にクラウド音声処理の確認が表示されます。

## LLM 最適化 (Beta) の設定

メニューバー → **LLM テキスト最適化 (Beta)** → **設定** — プリセットを選択するかカスタム追加し、API キーとモデル名を入力。ストリーミング出力はカプセル内でリアルタイムにプレビューされます。

組み込みプリセット:**OpenAI** / **Anthropic** / DeepSeek / Moonshot (Kimi) / Qwen / GLM / Yi / Groq / **Ollama(ローカル)** / カスタム。

デフォルトの system prompt はディクテーション後の整形向けに調整(同音異義語、誤認識された製品名・API 名、フィラー、句読点を修正)され、認識言語に応じて自動切替。独自の prompt で上書きも可能です。LLM テキスト最適化は、音声ではなく認識テキストを選択したプロバイダーへ送信します。
